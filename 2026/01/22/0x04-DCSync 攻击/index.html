DCSync攻击

DCSync是什么

为了保证所有域控制器（DC）上的数据一致，DC 之间会定期进行目录复制。复制的内容包括用户账户、组、密码哈希等敏感信息。

正常情况下，域控间会通过 Directory Replication Service（DRS） 服务的 GetNCChanges 接口发起数据同步请求

DCSync的原理就是模仿一个域控制器，从真实的域控制器中请求数据，例如用户的哈希。该功能最大的特点就是不用登陆域控制器，即可远程通过域数据同步复制的方式获得域控制器上的的数据。

默认情况下，以下组拥有复制权限：

  • ​Administrators（域管理员）
  • ​Domain Controllers（域控制器组）
  • ​Enterprise Domain Admins（企业域管理员）
  • 以及任何被显式授予了 “Replicating Directory Changes” 和 “Replicating Directory Changes All” 权限（nTSecurityDescriptor）的账户或组。

怎么打？

用Adfind查询可以打dcsync的账户

在拥有权限的账户上运行mimikatz.exe，导出所有账户的哈希

通过该方法可以在不登陆域控的情况下获得krbtgt的hash，后续可以利用这个hash制作黄金票据来进一步利用

其他利用

当我们获得域管的权限后，我们可以为普通用户添加一下几条ACE访问控制项，使其具有DCSync的操作权限

DS-Replication-Get-Changes（GUID:1131f6aa-9c07-11d1-f79f-00c04fc2dcd2）复制目录更改：允许用户请求域内数据的复制，但不包括所有秘密数据。

DS-Replication-Get-Changes-All（GUID:1131f6ad-9c07-11d1-f79f-00c04fc2dcd2）复制目录更改所有项：允许用户复制域中的所有数据，包括用户的密码哈希等敏感信息。这是DCSync能够成功的关键。

DS-Replication-Get-Changes-In-Filtered-Set（GUID:89e95b76-444d-4c62-991a-0facbeda640c）

在大多数环境中，授予前两个就足够了

如果在域控我们可以通过Active Directory用户和计算机的高级属性里面勾选复制目录更改和复制目录更改所有项这两个权限来为用户添加DCSync的操作权限
