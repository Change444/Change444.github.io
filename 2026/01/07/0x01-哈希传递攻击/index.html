哈希传递攻击

什么是哈希传递攻击

哈希传递攻击是基于NTLM认证的一种攻击方式。哈希传递攻击的利用前提是我们获得了某个用户的密码哈希值，但是解不开明文。这时我们可以利用利用了NTLM认证协议的一个设计特性：服务器在验证用户时，并不需要用户的明文密码，只需要验证用户提供的NTLM哈希值是否正确。在域环境中，大量计算机在安装时会使用相同的本地管理员账号和密码。因此，如果计算机的本地管理员账号密码相同，攻击者就能使用哈希传递攻击登录内网中的其他机器。

哈希传递攻击适用情况：

在工作组环境中：

  • Windows Vista 之前的机器，可以使用本地管理员组内用户进行攻击。
  • Windows Vista 之后的机器（打了kb2871997补丁），只能是administrator用户（rid=500）的哈希值才能进行哈希传递攻击，其他用户(包括管理员用户但是非administrator)也不能使用哈希传递攻击，会提示拒绝访问。

在域环境中：

  • 只能是域管理员组内用户(可以是域管理员组内非administrator用户)的哈希值才能进行哈希传递攻击，攻击成功后，可以访问域内任何一台机器

为什么在本地环境只能用rid=500的用户进行哈希传递攻击？

原因在于用户账户控制（UAC）。UAC将管理员分为两种令牌：完全管理员令牌和已过滤的管理员令牌（标准用户令牌）。在本地交互式登录时，如果用户是管理员，他们会在登录后获得两个令牌，而当他们需要执行管理员任务时，UAC会提示他们提升权限。这就是为什么我们明明是管理员，但是需要点击“管理员的身份运行”来运行某些程序的原因

但是，当从远程进行网络登录时（如通过WMI、PSEXEC等），情况有所不同。对于远程连接，如果用户是本地管理员组的成员，但又不是内置的RID 500账户（即内置管理员账户），那么即使该用户是本地管理员，系统也会为其分配一个已过滤的管理员令牌（即标准用户令牌），而不是完全管理员令牌。这就导致我们就算拥有本地管理员组的账号（非rid=500），但是远程登录上去仍然是用户权限。

但是这也有例外，通过远程桌面（RDP）进行远程连接时，用户可以进行交互式登录，此时UAC会正常提示，用户可以通过同意UAC提示来获得完全管理员令牌。

为什么域管理员组的账号却可以进行哈希传递攻击获得管理员权限？

当具有域用户帐户的用户远程登录Windows Vista计算机并且该用户是Administrators组的成员时，域用户将在远程计算机上以完全管理员访问令牌运行，并且该用户的UAC被禁用在该会话的远程计算机上。

绕过方法

如果HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\LocalAccountTokenFilterPolicy项存在(默认不存在)且配置为1，将授予来自管理员所有本地成员的远程连接完整的高完整性令牌。这意味着未过滤非RID 500帐户连接，并且可以成功传递哈希值！这个可以当做后门

获取hash

从上面可以看出，能否进行哈希传递攻击的关键在于是否能获取hash

在Windows中创建密码后，密码经过哈希化处理后存储在安全账户管理器(SAM)，本地安全机构子系统(LSASS)进程内存，凭据管理器(CredMan),Active Directory中的ntds.dit数据库或者其他地方。

当用户注销、重启、锁屏之后，windows会让winlogon显示登陆界面，接收用户的输入之后，会将将密码交付给lsass( Local Security Authority Service) 进程，这个进程会将明文密码加密成NTLM hash，再与SAM数据库(Security Account Manager) 里对应的用户密码做对比。

ntds.dit位于域控，用于存储域内用户的密码hash，这里先不讲。重点讲lsass和sam

LSASS

LSASS（本地安全机构子系统服务） 是Windows安全的核心进程，它负责所有本地和域的登录认证。当一个用户（无论是本地用户还是域用户）交互式登录（控制台、RDP）或通过网络（SMB、WinRM）登录一台计算机时，系统会验证其凭据（如NTLM哈希），并在验证成功后，由LSASS进程在内存中缓存该凭据的一个副本，以便用于后续的SSO（单点登录）和网络资源访问。

通过读取和解析LSASS进程的内存空间，可以获取其中的凭据

  1. 使用mimikatz直接抓取（需管理员权限）

  2. 使用Procdump转储后离线分析

SAM

本地用户账户的哈希存储在 ​C:\Windows\System32\config\SAM​​ 文件中。这个文件始终被系统锁定。SAM文件中的哈希（称为 ​NTLM Hash​）是使用一个​系统启动时随机生成的密钥​（称为 SysKey​ 或 BootKey）加密的。

核心在于：为了解密SAM，需要 SysKey​。而 SysKey​ 的一部分（或全部）存储在 HKLM\SYSTEM​ 注册表Hive中。因此，当你同时拥有了 SAM​ 和 SYSTEM 文件（或Hive），你就拥有了解密所有本地用户NTLM哈希所需的全部材料。

1.使用mimikatz（需system权限）

2.注册表导出sam文件（需管理员权限）

IPC$共享

  • ​ “$”符号：表示这是一个隐藏的管理共享，在普通浏览网络时不可见。

  • ​经典利用步骤：

    1. ​net use \\目标IP\IPC$ "密码" /user:"用户名" （建立认证连接）

    2. ​copy evil.exe \\目标IP\C$\windows\temp\​ （通过默认的C$管理共享上传文件）

    3. ​sc \\目标IP create EvilService binPath= "C:\...\evil.exe" （远程创建服务）

    4. ​sc \\目标IP start EvilService （远程启动服务，执行程序）
