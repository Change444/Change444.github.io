漏洞分析

鉴权分析

根据漏洞情报，漏洞路由位于/webroot/decision/nx/report/v9/largedataset/export/excel，根据代码可以发现，传入的参数由

​LargeDatasetExcelExportHandler类的静态实例的Handler方法接收

跟进看一下是怎么处理传入的参数的，跟进发现转到了另一个类的handle方法，这里转换了一下线程名，然后继续调用父类的handle进行处理

这里主要是处理资源初始化、GZIP 压缩、异常分类之类的，我们主要关注断点处的handleRequest方法

获取seesion，看样子应该需要要检查请求包中的sessionID来鉴权

漏洞点

回到LargeDatasetExcelExportHandler​的dohandle，根据sessionId获取模板信息，同时初始化一个Calculator​，这里的Calculator指的是帆软的计算引擎，负责公式求值、参数解析之类的，接着initCreator初始化导出数据创建器，这部分会设计到参数解析，然后执行导出，我们主要关注initCreator，参数解析这部分的代码

这部分就比较重要了，var2是我们的请求包，里面的参数可以由我们控制，关注var2参与了什么逻辑

  1. 从请求中获取 __parameters__​ 参数值（可能是 JSON 字符串），调用 getParamsMap(var5) 解析为 Map
  2. ​getEntity​ 解析 XML 格式的 params​ 参数，构建 LargeDatasetExcelExportJavaScript 对象
  3. 调用 this.dealParam(var11, var1, var7, var2, var4, var6)：处理参数

XML如何构造？解析出来会是什么

getEntity，获取请求包中的params​参数，并使用xml进行解析，getHTTPRequestParameter可以在query、header、body、cookie等位置获取参数params，这里先new了LargeDatasetExcelExportJavaScript，然后readXMLObject（var3），所以关注LargeDatasetExcelExportJavaScript中readXML的实现来了解xml的构造

这个方法首先调用了一个父类的readXML方法，然后判断有无LargeDatasetExcelExportJS​标签，如果有，从XML 节点中读取 exportFileName​、dsName​、colNames 三个属性，如果没有则设置为空，这段方法需要的xml大概长这样

然后查看父类readXML的逻辑，这里会检测标签是否有Parameters​，如果有会进入BaseXMLUtils.readParameters(var1);

跟进去，这个方法用于处理不同的参数类型

我们主要看Parameter参数是如何被处理的,大概流程就是BaseXMLUtils.readParameter​判断参数类型，若为Parameter类型，则调用

​BaseXMLUtils.readParameter​->XMLableReader.readXMLObject​->Parameter.readXML

将xml转到Parameter类来处理

如果标签名是 "Object"​ 或 "O"​，则调用 GeneralXMLTools.readObject(var1) 从该标签中读取一个 Java 对象，如果没有这两个标签，则只从标签中获取name，作为xml对象的名称。父类解析的xml大概长这样

跟进readObject​，这里用于从 XMLableReader 中读取一个对象。它根据 XML 元素上的属性t来决定对象的类型，然后调用 Object_Tokenizer.tokenizerObject 来实际构建对象

这里负责根据 XML 中的类型标识将 XML 内容解析为相应的 Java 对象，上面不是传进来一个t吗，举个例子，如果我传入的xml标签里面有这样一段

​<Object t="I">25</Object>​，当解析到该元素时，tokenizerObject​ 会被调用，var3​ 为 I​，然后进入 Integer 分支，返回一个int类型的25，并存入

​parameters数组

分析到这里就差不多了解xml的大概结构了，总结一下：

  1. ​getEntity​主要用于解析请求包中的params​传入的xml，获取LargeDatasetExcelExportJavaScript​，这个类的colNamesMap​，dsName​，filename从xml中解析获取，均可控
  2. ​LargeDatasetExcelExportJavaScript​还调用了其父类的readXML方法，获取了Parameter​对象，并存入了Parameters​数组，Parameter对象的数据类型和参数也可控

例如，我们传入了这样一个xml，那么通过getEntity​，我们将得到一个 LargeDatasetExcelExportJavaScript 对象，包含：

  • ​fileName = "Ch4nge"
  • ​dsName = "test"
  • ​colNames = {}
  • ​parameters​包含一个name="city"​ 和 value=25（int）的Parameter对象

进一步利用

回到initCreator，前面解析完xml后，会分别检查xml解析LargeDatasetExcelExportJavaScript对象的FileName和DsName，其中DsName不能为空，不然会抛出异常，另外还会检查一下SessionID，存到var11中，接着一股脑放入dealParam这个方法中处理

​dealParam​ 的核心作用是将多个来源的参数值合并，并处理公式和占位符，最终生成一个统一的参数映射。参数优先级为：__parameters__​ > functionParams 中的值 > xml解析Parameter中的值，为了不让Parameter被覆盖，前两个参数可以设置为空

该漏洞的sink点在Calculator#evalValue这里，通过传入Formula​类型的Parameter对象利用其执行sql(...)函数去RCE
