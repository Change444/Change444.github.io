JNDI注入分析

什么是JNDI

JNDI（Java Naming and Directory Interface，Java 命名和目录接口） 是 Java 提供的一种用于访问命名和目录服务的 API。它允许 Java 应用程序通过统一的接口访问各种命名和目录服务，例如 LDAP、DNS、RMI Registry、文件系统等。

jndi 在 jdk 里面支持以下四种服务

  • LDAP：轻量级目录访问协议
  • 通用对象请求代理架构(CORBA)；通用对象服务(COS)名称服务
  • Java 远程方法调用(RMI) 注册表
  • DNS 服务,

主要接口包括 <font style="color:rgb(64, 64, 64);">Context</font>、<font style="color:rgb(64, 64, 64);">InitialContext</font>、<font style="color:rgb(64, 64, 64);">DirContext</font> 等。

按我的理解，JNDI感觉有点像把rmi，ladp等服务封装在一起的大包，方便统一访问

结合LDAP

前言

LDAP（Lightweight Directory Access Protocol，轻量级目录访问协议） 是一种用于访问和管理目录服务的协议。目录服务是一种特殊的数据库，用于存储和组织信息（如用户、组织、设备等），并提供高效的查询和检索功能。

不仅仅在java中，在其他地方也有其存在，ldap有着一些独特的机制，例如索引，属性等，同时java对象在ldap中也有多种存储形式，其中就包括JNDI Reference

这个漏洞在jdk8u191才被修复

JDK高版本漏洞

高版本JDK在RMI和LDAP的trustURLCodebase都做了限制，从默认允许远程加载ObjectFactory变成了不允许。RMI是在6u132, 7u122, 8u113版本开始做了限制，LDAP是 11.0.1, 8u191, 7u201, 6u211版本开始做了限制。

所以修复后的JDK版本无法在不修改trustURLCodebase的情况下通过远程加载ObjectFactory类的方式去执行Java代码。

虽然无法再使用远程加载类，但绕过限制的方法也随之出现。目前公开常用的利用方法是通过Tomcat的org.apache.naming.factory.BeanFactory 工厂类去调用 javax.el.ELProcessor#eval方法或groovy.lang.GroovyShell#evaluate方法，还有通过LDAP的 javaSerializedData反序列化gadget

利用本地Factory类

调试一下可以发现，在原本的利用途中程序直到loadClass之前都能正常读到Reference类型指定的类，所以在8u191之后，Reference类型还是可以利用的，只是不能指向远程类了

我们可以令Reference对象指向本地存在漏洞的Factory类，该恶意 Factory 类必须实现 javax.naming.spi.ObjectFactory 接口，实现该接口的 getObjectInstance() 方法。

这里直接引用浅蓝师傅的解释

在Tomcat的<font style="color:rgb(255, 64, 129);">catalina.jar</font>中有一个<font style="color:rgb(255, 64, 129);">org.apache.naming.factory.BeanFactory</font>类，这个类会把<font style="color:rgb(255, 64, 129);">Reference</font>对象的<font style="color:rgb(255, 64, 129);">className</font>属性作为类名去调用无参构造方法实例化一个对象。然后再从<font style="color:rgb(255, 64, 129);">Reference</font>对象的Addrs参数集合中取得 AddrType 是 forceString 的 String 参数。

接着根据取到的 forceString 参数按照<font style="color:rgb(255, 64, 129);">,</font>逗号分割成多个要执行的方法。再按<font style="color:rgb(255, 64, 129);">=</font>等于号分割成 propName 和 param。

最后会根据 propName 作为方法名称去反射获取一个参数类型是 <font style="color:rgb(255, 64, 129);">String.class</font>的方法，并按照 param 从 Addrs 中取到的 String 对象作为参数去反射调用该方法。

而刚好<font style="color:rgb(255, 64, 129);">javax.el.ELProcessor#eval</font>和 <font style="color:rgb(255, 64, 129);">groovy.lang.GroovyShell#evaluate</font>这两个方法都是可以只传一个String参数就能够执行攻击代码，且依赖库比较常见所以被经常使用。
