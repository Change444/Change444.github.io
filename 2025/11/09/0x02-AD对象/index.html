AD对象

AD对象（Active Directory Object） 是存储在Active Directory数据库（NTDS.dit）中的一条记录，它代表网络中的一个具体实体，如用户、计算机、组、策略等。每个 Active Directory 对象都由唯一的 SID（安全标识符）标识，被用来允许或拒绝对域中各种资源的访问。

AD对象的本质是什么？

本质就是Active Directory数据库存储的条目，区分不同对象依靠对象的架构定义，类似java的类属性，不同类的属性不同，例如，user​ 类对象的架构定义了它必须有 sAMAccountName​ 属性，可以有 telephoneNumber​ 属性，但不能有 operatingSystem​ 属性（那是 computer 类的属性）。

每个对象有唯一的可分辨名称，反应它在目录树中的位置，如我们上图中李四在AD数据库的名称为：

​CN=李四，OU=行政部，DC=test，DC=internal

  • ​CN：通用名称（对象本身的名字）
  • ​OU：组织单元（容器）
  • ​DC：域组件（域的DNS名称部分）

这种结构的好处是可以实现管理委派（可以针对某个OU进行管理员授权）和策略应用（GPO可以链接到OU）。可以通过这张图来理解这种结构

对象的类型包括什么？

​用户（User）： 用户账号，存储着凭据（哈希）、组成员关系、登录时间等。

计算机（Computer）： 表示域内的主机

组（Group）：安全主体。它本身有一个SID，可以出现在其他对象（如文件、共享文件夹、GPO）的访问控制列表中，被“授予”权限。其中比较重要的组是域管理员（Domain Admin）

组织单位（OU）：容器对象。它本身不是安全主体（没有SID），不能直接被授予权限

组策略对象（GPO）

域（Domain） 和林（Forest）

‍

组和组织单元的区别：

  • 组（Group）是权限的集合。 它决定了 ​ “谁可以做什么” 。
  • 组织单元（OU）是对象的容器。 它决定了 “策略应用到谁身上”以及“谁来管理谁” 。

‍

用户（User）

我们来搜索一下objectClass=user对象，发现底下既有我们域内的计算机也有之前定义的用户账号，查看一下继承关系，发现computer其实是user的子类，所以机器用户其实本质就是域用户。所以，实际上在没有域账户的前提下将机器用户提权到system权限可以充当域用户来使用，进行域的横向渗透

来几个user类比较重要的属性：

userPrincipalName：指定要由客户端进行身份验证的服务的用户主体名称 (UPN)。用于标识账户名以及其所在域

sAMAccountName：存储帐户登录名或用户对象，相当于upn的前缀

userAccountControl：用于控制用户账户的各种设置。这个属性的值是一个数字，可以通过设置不同的位来启用或禁用特定的账户选项

在Active Directory中，UserAccountControl属性包含多个标志，这些标志代表了用户账户的不同设置。例如，如果要禁用一个账户，可以将UserAccountControl属性设置为​514​（十进制），这相当于将​ACCOUNTDISABLE​（值为2）和​NORMAL_ACCOUNT（值为512）这两个标志相加。这些标志是累积的，可以组合使用以设置多个选项。

以下是一些常见的UserAccountControl标志及其对应的十六进制和十进制值：

  • ​SCRIPT（0x0001 / 1）: 将运行登录脚本。
  • ​ACCOUNTDISABLE（0x0002 / 2）: 已禁用用户账户。
  • ​HOMEDIR_REQUIRED（0x0008 / 8）: 需要主文件夹。
  • ​PASSWD_NOTREQD（0x0020 / 32）: 无需密码。
  • ​PASSWD_CANT_CHANGE（无法直接设置）: 用户无法更改密码。
  • ​NORMAL_ACCOUNT（0x0200 / 512）: 表示典型用户的默认账户类型。
  • ​DONT_EXPIRE_PASSWORD（0x10000 / 65536）: 密码永不过期。
  • ​SMARTCARD_REQUIRED（0x40000 / 262144）: 需要使用智能卡登录。

这张图表示密码永不过期且账户正常的用户（65536+512），如果它被禁用了，那它的值将为66050（65536+512+2）

组（Group）

组（Group）是用户账号的集合用户账号的集合称为组。按照用途分类，组被分为安全组和通信组：

  1. 通讯组：给通信组发信息时，组内的用户都能收到，类似微信群组

  2. 安全组：对一个组赋予权限，则组内都是那个权限，安全组则是用户权限的集合。

这里主要将安全组，在域中，根据作用域不同，安全组又被划分为：

  1. 域本地组（Domain Local Group）
  2. 通用组（Universal Group）
  3. 全局组（Global Group）

域本地组（Domain Local Group）

主要用于分配权限，可以包含任何域的用户、全局组、通用组，甚至可以包含同一域的其他域本地组，但权限仅限于本域。

如果要查看域本地组，需要用到命令net localgroup /domain

Administrators：管理员组，改该组的成员可以不受限制地存取计算机/域的资源。不仅是最具有权限的一个组，也是在活动目录和域控制器中默认具有管理员权限的组，是域森林中强大的服务管理组。

Remote Desktop Users：远程登录组，处于远程登录组中的用户才有权限使用远程登录服务。

Print Operators：打印机操作员组，位于该组内的成员可以管理网络中的打印机，包括建立、管理和删除网络打印机，并可以在本地登录和关闭域控制器。

Account Operators：账号操作员组，该组的成员可以创建和管理该域中的用户和组并为其设置权限，也可以在本地登录域控制器，但是，不能更改属于 Administrators 或 Domain Admins 组的账户，也不能修改这些组。在默认情况下，该组中没有成员。

Server Operators：服务器操作员组，该组的成员可以管理域服务器，其权限包括建立/管理/删除任意服务器的共享目录、管理网络打印机、备份任何服务器的文件、格式化服务器硬盘、锁定服务器、变更服务器的系统时间、关闭域控制器等。在默认情况下，该组中没有成员。

BackupOperators，备份操作员组，改组的成员可以在域控制器中执行备份和还原操作，并可以在本地登录和关闭域控制器。在默认情况下，该组中没有成员。

全局组（Global Group）

主要用于组织同一域内的用户，可以包含同一域的用户和全局组，可以被其他域的组使用，但不能包含其他域的用户。可以在任何域中被授予权限

Domain Admins，管理员组，该组的成员在所有加入域的服务器、域控制器和活动目录中均默认拥有完整的管理员权限。因为该组会被添加到自己所在域的 Administrators 组中，因此可以继承 Administrators 组的所有权限。同时，该组默认会被添加到每台域成员计算机的本地 Administrators 组中，这样，Domain Admins 组就获得了域中所有计算机的所有权。如果希望某用户成为域系统管理员，建议将该用户添加到 Domain Admins 组中，而不要直接将该用户添加到 Administrators 组中。

Domain Users，域用户组，该组的成员中是所有的域用户。在默认情况下，任何由我们建立的更户账号都属于 Domain Users 组。因此，如果想让所有的账号都获得某种资源在取权限、可以将该权限指定给域用户组，或者让域用户组属于具有该权限的组。

Domain Computers，域成员主机组，该组的成员是域内所有的域成员主机，任何由我们建立的计算机账号都属于 Domain Computers 组。

Domain Controllers，域控制器组，该组成员包含了域内所有的域控制器。

Domain Guests，域访客用户组，该组内的成员默认为域访客用户，域成员计算机会自动将此组加到本地的 Guests 组中。

Group Policy Creator Owners：此组成员可以修改域的组策略。

DnsUpdateProxy：此组成员允许替其他客户端（如DHCP服务器）执行动态更新的 DNS 客户端。

Read-only Domain Controllers：此组中的成员是域中的只读域控制器

通用组（Universal Group）

适用于跨域权限分配，可以包含任何域的用户、全局组和其他通用组，可以在任何域中使用，但需要在全局编录中复制。

Enterprise Admins：企业系统管理员组，改组是域森林根域中的一个组。Enterprise Admins 组在域森林中的每个域内都是 Administrators 组的成员，因此对所有域控制器都有完全访问控制权。

Schema Admins：架构管理员组，是域森林根域中的一个组，可以修改活动目录和域森林的模式。Schema Admins 组是为活动目录和域控制器提供完整权限的域用户组，因此，该组成员的资格是非常重要的。

通过命令net group /domain可以看到全局组和通用组的成员

‍

关于域本地组Administrators​和全局组Domain Admins​的区别，他们之间是嵌套关系，因为当一个计算机（加入域时，Windows 会自动将域范围的 Domain Admins​ 组添加到该计算机的本地 Administrators​ 组中，这就解释了为什么全局组的作用域可以是多个不同的域
