域介绍

什么是Active Directory

Active Directory 是由微软开发的一种目录服务，主要用于 Windows 网络环境，用于集中管理网络中的用户、计算机、策略和其他资源。

为什么叫目录服务？答：类似电话薄，人的信息（姓名，地址，电话号码）作为一个条目存在电话薄中供查询，扩展到网络，用户、计算机、打印机被抽象为对象，被存储在像电话薄一样的服务中供用户管理和查询，这类服务就被称为目录服务。

目录服务的特点：

  1. 读多写少
  2. 为超快的读取和查询而极度优化
  3. 层次化、树状结构
  4. 基于开放标准LDAP
  5. 对象和属性

可以把Active Directory理解为一个网络电话薄

什么是域（Domain）

域是Active Directory的实例，是共享公共 Active Directory 数据库的所有 Active Directory 对象（如用户、计算机、组等）的集合或结构，并由称为域控制器的域的主服务器管理。域始终以其唯一的名称来引用，并且具有正确的域名结构。

域中包含的几个重要组件：

  • 组、用户、计算机等对象。
  • 身份认证服务
  • 组策略
  • DNS
  • DHCP

SID

SID（安全标识符） 是Windows系统中用于唯一标识一个安全主体的核心数据结构。它的结构为：域SID​ + -​ + RID，可以通过whoami /all来查看sid，一般情况下管理员的rid为500

域控制器（Domain Controller）

域控制器（Domain Controller, DC） 是 Active Directory 域服务（AD DS） 的运行载体，用于处理域成员的身份验证请求、统一身份存储、权限管理与策略分发、信任与关系管理等工作

与域管（Administrator）的关系：域控是运行Active Directory服务的主机，域管是拥有最高权限的账号，在域成员登录域管的账号可以远程管理域控，但凭据会暴露在成员机上

域树（Domain Tree）

域树表示为一系列以分层顺序连接在一起的域，这些域使用相同的DNS命名空间。当我们将子域添加到父域时，会创建域树。例如，有一个根域rootdse.lab，并向其添加了一个新的域矩阵(FQDN为matrix.rootdse.lab)，一旦在两者之间自动创建树系信任，它就会成为同一域树的一部分。

森林（Forest）

多个域树的集合，所有域通过信任连接在一起。林中的每个域都可以有一个或多个域控制器，这些域控制器可以与其他域交互，也可以访问来自其他域的资源。林的名称与根域相同。如果林包含单个域，则该域本身就是根域。

可通过Get-ADForest查看林的名称

信任（Trusts）

信任（Trust） 是两个 Active Directory 域（或林）之间的一种关系，它允许一个域中的用户被另一个域中的资源验证和授权。

信任可以是单向也可以是双向，信任可以传递。默认情况下，当添加子域或添加域树时，会自动创建双向可传递信任。
