域权限

windows访问控制模型

Active Directory权限控制依赖windows访问控制模型来实现，关于windows访问控制模型是什么，用大白话讲就是定义谁在什么条件下可以对什么执行操作， 这个以后再细学。

windows 访问控制模型是由两部分组成。

● 访问令牌（Access Token）包含用户的标识(User SID,Group SIDS)，以及特权列表。

● 安全描述符（security identifiers）。被访问的安全对象的相关安全信息。

访问模型的作用范围包括不限于：

● NTFS卷上的文件和目录

● 注册表项

● 网络共享

● 服务

● Active Directory对象

● 进程等等

‍

ACL（Access Control List）

它分为DACL（自由访问控制列表）和SACL（系统访问控制列表）

DACL的作用是定义哪个用户，或哪个用户所属的组可以对对象执行什么操作及其深度

SACL的作用是定义哪些访问尝试需要被记录到安全日志

‍

借用轩哥的文章来展示一下ACL的工作流程

  • A尝试访问D安全对象

  • 系统获取当前A访问线程的AccessToken以及相关信息，比如SID和各个groupSID

  • 信息获取完毕，开始按照顺序在安全对象D的DACL表中检查各个ACE（Deny​ACE的优先级大于Allow），通常有如下三种情况

    • 某一条拒绝类型的ACE显式拒绝令牌中某个受信者的所有访问权限 （X）
    • 检查完所有ACE但没有一个权限显式允许，那么系统会隐式拒绝该访问（默认拒绝X）
    • 一条或多条允许类型的ACE允许令牌中列出的受信者的所有访问权限（√）

总结就是

如果每一条ACE都没有命中，那么默认拒绝。如果有一条拒绝ACE命中，那么直接拒绝。

如果有一个，或者多个允许ACE命中，那么会累计允许的深度和数量，最终得出当前A线程拥有对当前安全对象的读、写、执行具体权限。

注意：如果一个安全对象没有DACL，系统就会授予该对象的所有访问权限给任何用户；如果一个安全对象有DACL，但是DACL里面没有ACE，那么系统会拒绝对该对象的任何访问。

‍

ACE（Access Control Entry）

看名字也能猜到，ACL是由一条条ACE构成的，他主要的任务是定义

● 谁对你有权限

● 是允许还是拒绝

● 有什么权限

● 这个权限能不能被继承

‍

SDDL（Security Descriptor Definition Language）

安全描述符定义语言（SDDL），微软官方的解释是将安全描述符描述为文本字符串的语言。nTSecurityDescriptor属性的值就是用SDDL表示的

SDDL 格式结构

SDDL 字符串通常包含以下几个部分：

  1. Owner (所有者)
  2. Group (组)
  3. DACL (Discretionary Access Control List， discretionary access control list)
  4. SACL (System Access Control List，系统访问控制列表)
