前言

什么是Commons BeanUtils

Apache Commons BeanUtils 是 Apache Commons 项目集的一个子项目，它提供了一组工具类，包含了属性拷贝、类型转换、属性获取和设置等功能，用于简化 JavaBean 的属性操作

什么是JavaBean

JavaBean 是一种遵循特定编写规范的 Java 类，通常用于表示应用程序的数据模型。

例如：

可以将JavaBean理解为一种规范

Commons BeanUtils利用点

<font style="color:rgb(35, 38, 59);">commons-beanutils</font>提供了一些方法能够调用符合JavaBean规范的类的方法，如：<font style="color:rgb(35, 38, 59);">PropertyUtils.getProperty</font>

通过这个方法，使用者可以很方便地调用任意对象的getter

因此，如果getter方法存在可以rce的点可以利用的话，就存在安全问题了。

链条分析

在加载字节码的时候，我们常常利用TemplatesImpl.newTransformer作为链条尾部的开始。观察可以发现TemplatesImpl符合JavaBean的规范。在这个类里有一个方法getOutputProperties调用了newTransformer。鉴于这一点，我们可以通过调用getOutputProperties来延长尾部链条

getOutputProperties是一个getter，还记得上面讲的Commons BeanUtils利用点吗，我们可以通过PropertyUtils.getProperty来调用TemplatesImpl.getOutputProperties

现在要找到哪个类调用了PropertyUtils.getProperty，我们看到BeanComparator.compare,javaBean和property都由用户指定，非常理想

接下来看谁调用了compare，这个好找，我们来看CC4，PriorityQueue这个类能调用任意类的compare方法，同时也是入口类，刚好能与BeanComparator连接
